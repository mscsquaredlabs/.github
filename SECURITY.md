We take the security of **mscsquared labs** projects seriously and appreciate responsible disclosure of vulnerabilities. This document explains how to report security issues and what you can expect from us.

## üì£ Reporting a Vulnerability

If you believe you have found a security vulnerability in this project, **please do not open a public issue**.

Instead, contact us privately:

- Email: **security@mscsquaredlabs.com**
- Subject line suggestion: `Security Report: <project-name>`

Please include as much detail as possible:

- A clear description of the issue
- Steps to reproduce
- Any proof-of-concept code or requests
- The affected version or commit (if known)
- Your environment details (OS, runtime, configuration) if relevant

We aim to acknowledge reports **within 72 hours** and provide regular updates on our progress towards a fix.

When we receive a security report, we will:

1. **Confirm receipt** of your report.
2. **Investigate** the issue and verify its impact.
3. **Develop and test a fix**, prioritizing issues by severity.
4. **Coordinate disclosure**, providing guidance on updating to a patched version.
5. **Credit contributors** who report vulnerabilities, if they wish to be acknowledged.

We appreciate your help in keeping our users and the wider ecosystem safe.

## üîç Responsible Disclosure

We kindly ask that you:

Thank you for helping to improve the security of **mscsquared labs** projects and the broader open-source community.
